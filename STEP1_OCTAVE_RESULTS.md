## § Verdict: Outcome B — Approximately Reproduces Tables 8–11 from Proper Initialization

**A (deterministic)**: No — 4/10 random starts failed to find the Tables 8–11 basin.

**B (approximate, dominant basin)**: Yes — 6/10 starts converged near the published solution.
P diagonals in the main-basin runs cluster around [0.977–0.993, 1.000–1.005, 0.968–0.978, 0.980–0.999],
close to the published [0.989, 1.001, 0.968, 0.994]. The LL of the best random start is within
0.08 units of the published value — essentially the same solution. The main basin is well-defined.

**C (multimodal, economically distinct)**: Partially — the degenerate basins exist (LL ≈ 1373–2137)
but are economically nonsensical (explosive eigenvalues on A or g, P diagonals far outside [0.5, 1.1]).
These are not "alternative economic interpretations"; they are numerical artifacts of a misidentified
objective. The published BCKM warm-start (in `runmleadj.m` x0a/x0b) avoids all of them.

**Conclusion**: The BCKM objective has a dominant, well-defined global basin corresponding to Tables
8–11. Proper initialization (BCKM's three warm-starts) reliably finds it. Random perturbations of
±10–20% can miss it (4/10 failure rate), confirming the landscape is non-convex. For our Python
pipeline, this means the warm-start strategy is critical — random restarts with large noise will waste
time in degenerate basins, while BCKM-faithful initialization converges correctly.

## § Anomalies / Flags for Synthesis

1. **Level-ratio f-stats**: BCKM's `runall.m` uses the level-ratio formula `(Y(t)/Y(t₀) - CF(t)/CF(t₀))²`,
   not log-deviation SSR. This is confirmed in the code. Our Python `phi_statistics` currently uses
   log-deviation SSR — this is the primary source of our f-stat disagreement. Needs to be addressed
   in the Python pipeline.

2. **Sample 1980Q1–2014Q4 (T=140)**: BCKM's stored estimation used `mlestart=1980.25`. Our Python
   pipeline's `us_1969_2014.parquet` targets 1969Q1 which only achieves T=152 (starting 1976Q1) due
   to LFWA data availability. The BCKM original used 1980Q1, not 1969Q1 — aligning the sample start
   to 1980Q1 is the correct approach and would also resolve the T mismatch.

3. **adjc=2 (BGG adjustment costs)**: The published results use adjustment-cost-augmented residuals
   (`adja=12.88`). Our Python model currently has no adjustment costs (`adja=0` equivalent). The
   effect on the f-stats is unknown without further testing, but it shifts the investment-wedge
   structural equations and likely affects fY[τ_x] most.

4. **`kfilter.m` is a steady-state Kalman filter** (DARE-based, constant gain K). This confirms
   the CLAUDE.md guidance that we should use DARE-per-call rather than a time-varying filter.
   The Python implementation already does this following the other machine's work.

5. **No bootstrap standard errors computed**: `runmleadj.m` has SE computation commented out.
   BCKM Table 8 reports no standard errors for P — only point estimates.
